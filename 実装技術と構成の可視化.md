このドキュメントは、現在の実装を「使用技術」「アプリ構成」「データフロー」「運用フロー」の観点で、Mermaid 図で把握するためのものです。

## 3) 日次バッチのデータフロー

```mermaid
sequenceDiagram
  participant WF as GitHub Actions(update.yml)
  participant Main as fetch_and_generate.py
  participant HN as HN/Algolia API
  participant LLM as OpenRouter API
  participant Gen as HTML/Sitemap Generator
  participant Docs as docs/
  participant Repo as GitHub Repository(main)
  participant Pages as Cloudflare Pages

  WF->>Main: python -m scripts.fetch_and_generate
  Main->>HN: トップ記事・コメント取得
  HN-->>Main: Story/Comment データ
  Main->>LLM: タイトル翻訳・コメント要約
  LLM-->>Main: 日本語テキスト
  Main->>Gen: DailyReport を渡してHTML生成
  Gen->>Docs: index/archive/about/privacy/sitemap等を書き出し
  WF->>Docs: pagefindで検索インデックス生成
  WF->>Repo: docs/ をコミット & rebase付きpush(最大3回)
  Repo->>Pages: デプロイ反映
```

## 4) GitHub Actions ワークフロー構成

```mermaid
flowchart TD
  S["schedule / workflow_dispatch"] --> C["concurrency: hn-daily-update-main"]
  C --> CO["actions/checkout@v4<br/>対象リポジトリ"]
  CO --> CORE["actions/checkout@v4<br/>hn-matome-core を core/ 配下へ"]
  CORE --> PY["actions/setup-python@v5 (3.11)"]
  PY --> PIP["pip install -r requirements.txt"]
  PIP --> RUN["cd core && python -m scripts.fetch_and_generate"]
  RUN --> SYNC["core/docs -> docs へ同期"]
  SYNC --> PF["npx pagefind --site docs"]
  PF --> COMMIT["git add docs && git commit"]
  COMMIT --> PUSH["fetch + rebase + push を3回リトライ"]
```

## 5) データモデル（実装ベース）

```mermaid
classDiagram
  class Comment {
    +int id
    +str author
    +str text
    +from_api(data) Comment
  }

  class Story {
    +int rank
    +int id
    +str title_en
    +str title_ja
    +str|None url
    +str hn_url
    +int score
    +int comment_count
    +datetime posted_at
    +list~Comment~ comments
    +str summary_ja
    +from_api(data, rank) Story
  }

  class DailyReport {
    +datetime date
    +list~Story~ stories
    +date_str str
    +date_ja str
  }

  DailyReport "1" --> "*" Story
  Story "1" --> "*" Comment
```
