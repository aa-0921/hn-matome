# GitHub Actions cron 定期実行 — 本当に使えるアイデア集

> 前提: Webサービス（グローバルツール）で月5,000円→5万円を目指している。
> note / Qiita / Zenn / X にコンテンツを投稿してフォロワー・告知力を増やしている最中。
> 「直接稼ぐ」だけでなく、「稼ぐための布石になる」ものも対象。

---

## なぜ cron 定期実行が強いのか（再確認）

- public リポジトリなら **月額0円で無制限**
- 最小5分間隔。VPS不要。GitHub アカウントだけで動く
- 2026年3月〜: `timezone: "Asia/Tokyo"` で JST 指定可能に
- **60日コミットなしで停止**する点だけ注意

---

## 前回の調査で何がダメだったか

前回のアイデア（AI自動ブログ、価格追跡サイト、死活監視、テンプレ販売等）は
**「cron でできること」から逆算して考えてしまった**ため、以下の問題があった：

- それ単体で稼ぐには競合が多すぎる or 市場が小さい
- 自分の既存戦略（Webツール × コンテンツ投稿 × フォロワー増）と繋がらない
- 「やればいい」で終わり、再現性の根拠がない

今回は **「自分の戦略にどう組み込むか」** という視点で整理し直す。

## 2つの軸で整理

### 軸1: 布石系（直接稼がないが、告知力・信頼・流入を増やす）
### 軸2: 収益直結系（再現性のあるパターンに限定）

---

## 軸1: 布石系 — 告知力・信頼・流入を自動で積み上げる

### 1-A. 過去記事の自動リサイクル投稿（X / SNS）

**何をするか**: 過去に書いた note / Zenn / Qiita の記事をランダムに選び、AIでキャッチーな宣伝文を生成して X に自動投稿する。1日2回。

**なぜ効くか**:
- 個人開発者のコンテンツは「投稿直後」しか見られない。過去記事の再浮上で新規フォロワーへの露出を増やせる
- 手動でやると面倒すぎて続かないが、cron なら永久に回る
- X API 無料枠は月500ポストまで対応（1日2回 = 月60ポストなら余裕）

**実装の難易度**: ★☆☆☆☆（Python + tweepy + AI API で数時間）

**実例**: [ideaman's Notes](http://notes.ideamans.com/posts/2025/x-ai-posting.html) が実装・運用して効果を報告

**あなたの戦略との接続**:
```
Zenn/Qiita/noteに記事を書く（手動・既存の取り組み）
    ↓
cron が1日2回、過去記事をランダム選出
    ↓
AI が宣伝文を自動生成
    ↓
X に自動投稿（OGP画像付き）
    ↓
フォロワーが増える → プロダクトローンチ時の告知力が上がる
```

---

### 1-B. Build in Public 自動化（開発進捗の自動共有）

**何をするか**: GitHub のコミットログから、その日の開発進捗を AI が要約 → X / note に自動投稿。

**なぜ効くか**:
- Build in Public をやっているインディーハッカーは、**ローンチ前に500〜5,000人の「応援フォロワー」を獲得**している（海外事例多数）
- ある開発者は BIP で2,400フォロワーを獲得し、**ローンチ初日に $8,000 MRR** を達成
- 手動で毎日進捗を書くのは面倒 → 自動化すれば「書かないリスク」がなくなる

**ツール**: [LaunchLogs](https://www.launchlogs.com/) / [BuiltPublic](https://builtpublic.com/) が既に存在。自作も容易。

**実装の難易度**: ★★☆☆☆

**あなたの戦略との接続**:
```
Webツールを開発する（手動・既存の計画）
    ↓
cron が毎日、当日のコミットを AI で要約
    ↓
「今日やったこと」をXに自動投稿
    ↓
開発過程を見てくれるフォロワーが増える
    ↓
ローンチ時に「あのツール完成したんだ！」と反応してもらえる
```

### 1-C. GitHub プロフィール README の動的自動更新

**何をするか**: GitHub プロフィール README に「最新のZenn/Qiita記事」「開発中プロジェクトの進捗」「今月の草グラフ」を毎日自動反映。

**なぜ効くか**:
- GitHubプロフィールは **開発者の名刺**。ここが常に最新だと、プロダクトページやプロフィールを見に来た人の信頼が上がる
- 「この人、毎日ちゃんと活動してるんだ」という印象 = 有料プランへの転換率に間接的に効く

**実装の難易度**: ★☆☆☆☆（既存ツール多数。[github-readme-stats](https://github.com/anuraghazra/github-readme-stats) 78.9k stars）

---

### 1-D. Awesome List の自動メンテナンス（ニッチ領域で権威を取る）

**何をするか**: 自分のドメインに関連する awesome-list を作り、cron で新しいリポジトリやツールを自動検出 → PR を自動作成。

**なぜ効くか**:
- awesome-list は GitHub で **最もスターを集めやすいリポジトリ形態**（上位リストは数万〜数十万スター）
- [TrackAwesomeList](https://www.trackawesomelist.com/) は500以上のリストを追跡、600 stars
- 特定ニッチの「情報のハブ」になれば、そこからプロダクトへの導線を作れる
- 例: 「awesome-nisa-tools」「awesome-web-tools-for-creators」

**実装の難易度**: ★★☆☆☆

**あなたの戦略との接続**:
```
自分が作るWebツールと同じカテゴリの awesome-list を先に作る
    ↓
cron で新ツールを自動検出 → リストが常に最新
    ↓
リストにスターが集まる → GitHub 上での認知が広がる
    ↓
リストの中に自分のプロダクトも掲載 → 自然な導線
```

### 1-E. マルチプラットフォーム同時投稿（Zenn → Qiita 自動同期）

**何をするか**: Zenn にMarkdownで記事を書く → git push → cron で Qiita にも自動同期。

**なぜ効くか**:
- 同じ記事を2つのプラットフォームに載せるだけで **読者の母数が2倍**になる
- Zenn は GitHub 連携が公式、Qiita も API あり。note は API がないため非対応
- 既に多数の実装例あり（再現性◎）

**実装の難易度**: ★★☆☆☆

**参考**: [zenn.dev/long910 - マルチプラットフォーム自動同期](https://zenn.dev/long910/articles/2026-03-15-multi-platform-blog-strategy)

## 軸2: 収益直結系 — 再現性のあるパターン

### 2-A. プログラマティック SEO サイト（データ集約 × 自動更新）

**何をするか**: 特定ニッチの公開データ（API / RSS）を毎日自動収集 → ページを自動生成 → GitHub Pages で無料ホスティング。

**なぜ「再現性がある」と言えるか**:
- 個人開発者「アフィサーチ」（アフィリエイト案件検索サイト、約1万件DB）が ASP 成果報酬で収益化に成功（[Indieverse記事](https://indieverse.co.jp/articles/launch-database-seo-site/)）
- 海外では「Product Huntのデータを毎日スクレイピング → 10言語にAI翻訳 → 月間20万クリック」の事例あり（[Indie Hackers記事](https://www.indiehackers.com/post/342edb179d)）
- 別の開発者は自作SEOソフトで「1日4〜5記事自動公開 → 月$5,000の広告収入」（[Indie Hackers記事](https://www.indiehackers.com/post/case-study-making-5k-mo-from-my-blog-using-my-own-seo-content-software-ba395c1b60)）
- 個人開発のSEO×サブスクで「最高月収3.7万円で黒字化」（[Qiita記事](https://qiita.com/tomada/items/b06187360af9473480fc)）

**重要な成功条件**:
- テクニカルSEO以上に **「どの市場を選ぶか」が全てを決める**
- 「テンプレで大量ページ生成」はGoogleに薄いコンテンツとして弾かれる → 各ページに固有の価値が必要
- 構造化データ（JSON-LD）、内部リンク設計、サイトマップの自動生成を最初からやる

**あなたの戦略との接続**（例: NISA関連データサイト）:
```
cron で毎日:
  投資信託の運用実績データを公開APIから取得
  信託報酬の比較表を自動更新
  「今月のパフォーマンス上位ファンド」ページを自動生成
    ↓
GitHub Pages で無料ホスティング（月額0円）
    ↓
「NISA おすすめ」「投資信託 比較」で SEO 流入
    ↓
証券口座アフィリエイト（1件3,000〜20,000円）
```

**実装の難易度**: ★★★☆☆（データ取得は簡単だが、SEO設計に知識が要る）

### 2-B. 自分のWebツールの「鮮度」を自動で保つ

**何をするか**: 自分が開発するWebツールのコンテンツ・データを cron で自動更新し、常に「最新」の状態を維持する。

**なぜ効くか**:
- Googleは **「freshness」を検索ランキング要因の一つ**としている
- 個人開発のWebツールは「作って放置」になりがちだが、データが自動で更新されていれば SEO 上有利
- ユーザーが訪問したとき「最終更新: 今日」と表示されるだけで信頼が違う
- はてなブログの更新日を自動で最新にするツールだけで**検索順位上昇の効果が複数報告**されている

**具体例**:
- ツールで使うサンプルデータ・デモデータの自動更新
- 「対応フォーマット一覧」「FAQ」などの自動最新化
- ツールの利用統計（「累計○万回利用されました」）の自動更新
- ランディングページの「最新のユーザーレビュー」自動表示

**実装の難易度**: ★★☆☆☆

---

### 2-C. 定期データ取得 → コンテンツのネタにする（note / Zenn 記事量産）

**何をするか**: cron で毎週データを自動収集 → 「今週の○○ランキング」「先月の○○トレンド」のような定型記事の下書きを自動生成。

**なぜ効くか**:
- 「定期レポート型コンテンツ」は **毎回ゼロから企画する必要がない**（再現性が高い）
- 「週刊○○ニュース」「月間○○ランキング」のような連載は、単発記事より**読者の定着率が高い**
- データ収集部分を自動化すれば、人間は「考察を書く」だけで済む
- 金融バックグラウンドを活かすなら: 「今週のNISA人気ファンドTOP10」「今月の投資信託パフォーマンス比較」

**あなたの戦略との接続**:
```
cron が毎週月曜に実行:
  ① 公開APIからデータ取得（株価、ファンド実績、etc.）
  ② AI が「今週のハイライト」の下書きを生成
  ③ PR を自動作成 or ローカルファイルに保存
    ↓
あなたがレビュー・加筆（30分〜1時間）
    ↓
note / Zenn に投稿（手動 or 1-E の自動投稿）
    ↓
「毎週必ず出る連載」として認知される → フォロワー定着
```

**実装の難易度**: ★★☆☆☆

---

### 2-D. OGP画像の自動生成（SNSシェア率の底上げ）

**何をするか**: 記事を書いたら、タイトルから自動でOGP画像（SNSでシェアされたとき表示されるサムネイル）を生成する。

**なぜ効くか**:
- OGP画像がない記事は **X のタイムラインで目立たない → クリック率が低い**
- 手動で毎回作ると1枚15〜20分。自動化すれば数秒
- 統一デザインのOGP画像は **個人ブランドの認知** にも効く

**実装**:
- HTML+CSSテンプレート → Playwright でスクリーンショット → 画像として保存
- 記事のPR / push をトリガーに GitHub Actions で自動実行
- [zenn.dev/secure_auto_lab の実装例](https://zenn.dev/secure_auto_lab/articles/article-publisher-auto-ogp-and-note-api)

**実装の難易度**: ★★☆☆☆

## おすすめ実行順序

### 今日やる（30分〜2時間）
1. **1-A: 過去記事リサイクル投稿** — 最も手軽で、既存のコンテンツ資産をすぐ活かせる

### 今週やる（半日〜1日）
2. **1-E: Zenn → Qiita 自動同期** — 記事投稿の効果を2倍にする
3. **1-B: Build in Public 自動化** — プロダクト開発開始と同時にセットアップ

### 今月やる（開発と並行）
4. **2-C: 定期レポート型コンテンツ** — 「毎週出る連載」で読者を定着させる
5. **2-B: 自分のWebツールの鮮度維持** — ツール完成後に組み込む

### 中期（1〜3ヶ月）
6. **2-A: プログラマティックSEOサイト** — 金融データ集約サイトを検討（信用金庫経験が活きる）

## 共通の注意事項

| 項目 | 内容 |
|------|------|
| **60日ルール** | 60日間コミットがないリポジトリは cron 停止。月1回は何かコミットする |
| **cron の精度** | 数分〜十数分ズレる。正確な時刻が必要なら `workflow_dispatch` 併用 |
| **API キー管理** | GitHub Secrets に保存。コードにハードコードしない |
| **X API 制限** | 無料枠: 月500ポスト。1日2回投稿なら月60ポストで余裕 |
| **スクレイピング規約** | 対象サイトの robots.txt と利用規約を確認。APIがあればAPI優先 |
| **public 必須** | 無料で使うには public リポジトリ。Secrets はリポジトリ設定から入れれば安全 |

## まとめ: cron は「手間をゼロにする仕組み」として最強

cron 定期実行の本質は「稼ぐ仕組み」ではなく、**「やった方がいいけど面倒で続かないこと」を自動化する仕組み**。

あなたの戦略（Webツール開発 × コンテンツ投稿 × フォロワー増）の中で、
手動だと「三日坊主」になりがちな部分を cron に任せることで、
**人間は「記事を書く」「ツールを作る」に集中できる**。

```
稼ぐ（Webツール × サブスク/広告）
  ↑ これは人間がやる
  │
告知する（X / note / Zenn / Qiita）
  ↑ ここを cron で自動化・効率化する
  │
信頼を積む（毎日の活動の可視化・コンテンツの鮮度）
  ↑ ここも cron で自動化する
```
