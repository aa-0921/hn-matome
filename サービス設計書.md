## サービス概要

| 項目 | 内容 |
|---|---|
| サービス名 | HackerNews 日本語まとめ & AI要約 |
| ターゲット | Hacker News を知っている / 興味ある日本人エンジニア |
| 提供価値 | 英語が苦手でも HN の話題・英語コミュニティの反応を毎日日本語で読める |

## アーキテクチャ

```
HN公式API（Firebase）
  ↓
GitHub Actions cron（毎日 JST 8:00）
  ↓
Python スクリプト
  ├── トップ30記事のタイトル取得・日本語翻訳
  ├── 各記事の上位5コメント取得・日本語要約
  └── 静的 HTML 生成
  ↓
GitHub Pages（公開）
```

## MVP スコープ

### 含めるもの

- [ ] HN Top 30 記事のタイトル日本語翻訳
- [ ] 各記事のスコア・コメント数・投稿時刻表示
- [ ] 上位 5 コメントの日本語要約
- [ ] 日付別アーカイブ（過去記事を日付から閲覧）
- [ ] レスポンシブ対応（モバイル）

### 含めないもの（後回し）

- 元記事本文の要約（外部 fetch が必要・コスト高）
- ユーザー登録・パーソナライズ
- ニュースレター配信
- Reddit 統合

## データ取得仕様

### HN API エンドポイント

```
# トップ記事 ID 一覧
GET https://hacker-news.firebaseio.com/v0/topstories.json

# 記事詳細
GET https://hacker-news.firebaseio.com/v0/item/{id}.json

# コメント詳細（同じエンドポイント）
GET https://hacker-news.firebaseio.com/v0/item/{comment_id}.json
```

### 取得フィールド

| フィールド | 用途 |
|---|---|
| `title` | 日本語翻訳対象 |
| `url` | 記事リンク（HN ページへ） |
| `score` | スコア表示 |
| `descendants` | コメント数 |
| `time` | 投稿時刻 |
| `kids[0..4]` | 上位5コメントの ID |
| コメント `text` | AI 要約対象 |

## LLM 利用設計

### 使用モデル

- OpenRouter 無料枠: `deepseek/deepseek-r1` 等
- 1日あたりのリクエスト数: 30（タイトル翻訳） + 30×5（コメント要約） = 最大 180 req
- OpenRouter 無料枠: 50 req/日 → **足りない**
  - 対策1: コメント要約をまとめて1リクエストにする（30記事分まとめて）
  - 対策2: タイトルも一括翻訳（1リクエストで30件）
  - → **実質 2〜5 req/日 に圧縮可能**

### プロンプト設計（タイトル一括翻訳）

```
以下の英語タイトルを自然な日本語に翻訳してください。
番号はそのまま保持し、技術用語はカタカナまたは原語で残してください。

1. {title_1}
2. {title_2}
...
30. {title_30}
```

### プロンプト設計（コメント要約）

```
以下は Hacker News の記事「{title}」に対する英語コメントです。
コメント全体を読み、議論の要点を日本語で200字以内にまとめてください。

{comments}
```

## GitHub Actions 設定

```yaml
# .github/workflows/update.yml
name: Update HN Daily

on:
  schedule:
    - cron: '0 23 * * *'  # JST 8:00 = UTC 23:00
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python scripts/fetch_and_translate.py
        env:
          OPENROUTER_API_KEY: ${{ secrets.OPENROUTER_API_KEY }}
      - run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add docs/
          git diff --cached --quiet || git commit -m "Update: $(date +%Y-%m-%d)"
          git push
```

## リスク

| リスク | 対策 |
|---|---|
| OpenRouter 無料枠の変更 | 一括リクエストで節約。有料化しても月数百円以内 |
| GitHub Pages の帯域制限 | ソフト制限 100GB/月。当初は問題なし |
| Y Combinator 商標クレーム | 「HackerNews」をサービス名に含むリスクあり。フォールバック名を検討 |
| 翻訳精度 | 誤訳免責事項を footer に記載 |
